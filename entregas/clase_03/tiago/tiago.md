# Definición propia de TDA.
Un TDA es un objeto que puede definir las operaciones y comportamiento según el caso, haciéndolo variable.

# Diferencia entre interfaz e implementación
La interfaz es lo que vemos de inmediato, y que ocupamos sin necesidad de entender de fondo. La implementación es el código detrás que hace que funcione.

# Problemas de pilas
Usaría una pila en problemas como historial de navegación, registros de acciones para el control z, o balanceo de paréntesis. O sistemas de inventario en juegos, como las macetas de barro del minecraft.
