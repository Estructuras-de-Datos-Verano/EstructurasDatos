- Mi definición de TDA: Es un concepto de un dato abstracto donde la información es en manera de pila, donde los datos tienen cierto orden.

- La interfaz es todo con lo que el usuario interactua para agregar/eliminar información y la implementación es todo lo que sucede de fondo para que la interfaz funciones, pero no necesitas conocer la implementación para poder utilizar la interfaz

- Lo ocuparia en problemas que necesiten diseñar un algoritmo que facilmente pueda deshacer los ultimos pasos sin modificar cosas, más que lo ultimo

## Interfaz

La interfaz propuesta esta en otro documento nombraso interfaz_pila en esta misma carpeta

- Propuesta: Ocupar clases de POO para proponer una interfaz para que puedas revisar los ultimos pasos que realizaste en un algoritmo

- Para verificar que las operaciones funcionen como se debe, basta con probar con una pila de dos elementos:

        1.- Empy: deberia de marcar que no lo esta ya que si tiene elementos
        2.- Pop: deberia de elimir el elemento de la parte superior y dejarnos unicamente el primer elemento
        3.- Push: deberia de agregar un tercer elemento de la pila en la parte superior
        4.- Peek: deberia de indicarte cual es el elemento superior de la pila
    Si al realizar una prueba las cosas no marcan lo que est indicado, entonces hay un error a la hora de programar y deberiamos crear raise errors para evitarlos o probar una vez más con Try Except
