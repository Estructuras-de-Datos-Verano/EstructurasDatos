## Recurso 7

- Tipo: Video
- Titulo: Estructuras de Datos | Primeros Pasos
- Autor/canal: Chio Code
- Idioma: Español
- Duracion si aplica: 10:55
- Enlace: https://www.youtube.com/watch?v=Df-sgxGzyTg
- Tema principal: Estructuras de datos lineales estáticas y dinámicas, y su manejo directo en la memoria RAM.
- Nivel: Básico / Intermedio
- Que explica bien: La diferencia física en la memoria de la computadora entre un arreglo (que guarda datos en casillas continuas) y una lista enlazada (que guarda nodos dispersos conectados por apuntadores).
- Que fue confuso: Menciona rápidamente ejemplos de estructuras no lineales complejas (como grafos y montículos) sin profundizar, lo que puede sentirse abrumador si es la primera vez que las escuchas.
- Que ejemplo fue util: Explicar que el acceso a un índice en un arreglo es de tiempo constante $O(1)$ porque para la computadora es simplemente hacer una multiplicación matemática para encontrar la casilla correcta.
- Como se relaciona con el curso: Refuerza la justificación matemática de por qué, en el ejercicio de encontrar el "Dato más pequeño", la representación ideal en memoria es simplemente un arreglo y una variable, demostrando cómo funciona físicamente detrás del código.
- Lo recomendarias: Sí, es excelente para estudiantes de ciencias exactas (como matemáticas aplicadas) porque ayuda a entender la lógica detrás de cómo las computadoras gestionan el espacio físico.

## Recurso 8

- Tipo: Documentación Oficial / Texto Escrito
- Titulo: 5. Estructuras de datos (Tutorial de Python)
- Autor/canal: Python Software Foundation
- Idioma: Español
- Duracion si aplica: N/A (Tiempo estimado de lectura: 20-25 minutos)
- Enlace: https://docs.python.org/es/3/tutorial/datastructures.html
- Tema principal: Uso, métodos y optimización de las estructuras de datos nativas de Python (listas, tuplas, conjuntos y diccionarios).
- Nivel: Intermedio
- Que explica bien: Las operaciones avanzadas con listas, cómo usarlas eficientemente como pilas (Stacks), por qué son ineficientes como colas (Queues) tradicionales y cómo solucionar esto usando `collections.deque`.
- Que fue confuso: La sintaxis de las listas de comprensión anidadas (nested list comprehensions) puede llegar a ser muy abstracta y enrevesada de leer si no estás muy familiarizado con álgebra de conjuntos o bucles múltiples.
- Que ejemplo fue util: El ejemplo de código que demuestra cómo implementar una cola rápida usando `from collections import deque`, explicando de forma clara el impacto en el rendimiento al añadir elementos por los extremos.
- Como se relaciona con el curso: Es la "fuente de verdad" del lenguaje de programación del curso. Les ayuda a entender qué métodos de Python ya están optimizados internamente para que no tengan que reinventar la rueda al diseñar sus algoritmos.
- Lo recomendarias: Sí, es de lectura obligatoria para cualquier estudiante que use Python para resolver problemas de ciencias de la computación.
- Pregunta tecnica: ¿Por qué la documentación oficial desaconseja usar una lista común de Python (`list`) para realizar operaciones de cola (Queue) donde se inserten o extraigan elementos desde el principio de la secuencia?
