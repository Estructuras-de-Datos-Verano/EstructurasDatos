Bloque 2: medir tiempos

Meta

Comparar operaciones con datos de distintos tamaños.

Cuidado

Una medición aislada puede mentir por ruido, caché, carga del sistema o tamaño muy pequeño.

$$\text{medición útil} = \text{repeticiones} + \text{comparación} + \text{interpretación}$$

time.perf_counter

import time

inicio = time.perf_counter()

# operación que queremos medir
resultado = 123 in datos

fin = time.perf_counter()
tiempo = fin - inicio

Idea

Tomamos una marca de tiempo antes y después de la operación.

La diferencia aproxima cuánto tardó.

Una función para medir

import time

def medir_tiempo(funcion, repeticiones=10):
    tiempos = []

    for _ in range(repeticiones):
        inicio = time.perf_counter()
        funcion()
        fin = time.perf_counter()
        tiempos.append(fin - inicio)

    return sum(tiempos) / len(tiempos)

¿Por qué usar repeticiones?

Ruido

La computadora hace otras cosas mientras medimos.

Escala

Operaciones muy pequeñas pueden verse casi instantáneas.

Promedio

Varias mediciones dan una señal más estable.

No buscamos segundos universales. Buscamos patrones de crecimiento.

Conjuntos

Buenos para pertenencia

Diccionarios

conteos = {}

for dato in datos:
    if dato not in conteos:
        conteos[dato] = 0
    conteos[dato] += 1

Pares clave-valor

$$\text{clave} \mapsto \text{valor}$$

Sirven para frecuencias, relaciones, etiquetas, costos y muchas representaciones matemáticas.

Counter

from collections import Counter

conteos.most_common(5)
