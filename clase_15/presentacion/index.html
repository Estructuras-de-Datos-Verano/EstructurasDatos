Relajar una arista

Intentar mejorar la mejor distancia conocida usando una arista.

candidata = distancia[u] + peso(u,v)

si candidata < distancia[v]:
    distancia[v] = candidata
    predecesor[v] = u

B=4, B→D=2, D=10: candidata=6 y 6<10, así que D mejora.

Interpretación

Distancia = prioridad.

Nodo = elemento.

Sale primero `(3,C)`.

Menor distancia acumulada significa mayor prioridad de procesamiento.

Ahora sí: Dijkstra

Calcula caminos mínimos desde un origen en grafos con pesos no negativos.

extraer (distancia_extraida, actual)
si distancia_extraida != distancia[actual]:
    continuar  # entrada obsoleta
para cada (vecino, peso):
    candidata = distancia[actual] + peso
    si candidata < distancia[vecino]:
        actualizar e insertar nueva candidatura
Una entrada vieja puede permanecer en el heap y se ignora al extraerla.

Min-heap

Reconstruir el camino

E ← D ← B ← C ← A

invertir

A → C → B → D → E

Los predecesores apuntan hacia atrás. Un destino inalcanzable produce camino vacío.

Complejidad

Tiempo

O((V+E) log V)

Las operaciones del heap aportan el factor logarítmico.

Espacio

O(V+E)

Grafo, tablas y candidaturas.

Buscar linealmente el mínimo pendiente conduce a O(V²).

Frontera de aplicación

A → B = 2
A → C = 5
C → B = -10

La ruta futura A→C→B cuesta -5 y puede mejorar un nodo que parecía listo.

Dijkstra requiere pesos no negativos; la implementación debe rechazar los negativos.
