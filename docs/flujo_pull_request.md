# Flujo de pull request

El alcance de una entrega debe ser verificable a partir de su ruta
`entregas/<alumno>/clase_XX/`.

Antes de abrir una revisión:

1. confirma la clase y el identificador del alumno;
2. ejecuta las pruebas públicas con `scripts/evaluar.py`;
3. ejecuta las pruebas propias cuando la práctica las solicite;
4. conserva la salida completa en el archivo indicado por la clase;
5. revisa que el conjunto de cambios no incluya materiales de otros alumnos;
6. sigue el formato de revisión definido en la práctica correspondiente.

Los índices de `docs/entregas_por_clase/` sirven para navegar, pero no se
copian dentro de una entrega ni sustituyen la evidencia técnica.
