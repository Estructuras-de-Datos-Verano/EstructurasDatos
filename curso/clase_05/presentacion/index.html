Casos de borde

Cola vacía

`desencolar()` debe lanzar `IndexError`.

Sin frente

`frente()` también debe lanzar `IndexError`.

Consulta

`frente()` no debe cambiar el tamaño.

Mini simulador FIFO

Frente: vacía · Tamaño: 0

Implementación 1: ColaLista

class ColaLista:
    def __init__(self):
        self._datos = []

    def encolar(self, elemento):
        self._datos.append(elemento)

    def desencolar(self):
        if self.esta_vacia():
            raise IndexError("cola vacía")
        return self._datos.pop(0)

¿Por qué mirar `pop(0)`?

Idea

Si quitamos el primer elemento de una lista, los demás deben moverse una posición.

["A", "B", "C", "D"]
pop(0)
["B", "C", "D"]

# B, C y D cambian de índice

Costo intuitivo

`append`

Agregar al final suele ser eficiente.

`pop(0)`

Puede requerir mover muchos elementos.

Decisión

La implementación importa cuando una operación se repite mucho.

Implementación 2: ColaDeque

from collections import deque

class ColaDeque:
    def __init__(self):
        self._datos = deque()

    def encolar(self, elemento):
        self._datos.append(elemento)

    def desencolar(self):
        if self.esta_vacia():
            raise IndexError("cola vacía")
        return self._datos.popleft()

Comparación rápida

ImplementaciónClaridadDesencolar muchas vecesUso recomendado
`ColaLista`Muy claraPuede ser costosaAprender y ejemplos pequeños
`ColaDeque`Clara al conocer `deque`Más adecuadaColas reales en Python

De `assert` a `pytest`

def test_desencolar_respeta_fifo():
    cola = ColaLista()
    cola.encolar("primero")
    cola.encolar("segundo")

    assert cola.desencolar() == "primero"
    assert cola.desencolar() == "segundo"

Probar errores esperados

Una prueba, dos implementaciones

IMPLEMENTACIONES = [ColaLista, ColaDeque]

@pytest.mark.parametrize("ClaseCola", IMPLEMENTACIONES)
def test_frente_no_elimina(ClaseCola):
    cola = ClaseCola()
    cola.encolar(10)
    cola.encolar(20)

    assert cola.frente() == 10
    assert cola.tamano() == 2
