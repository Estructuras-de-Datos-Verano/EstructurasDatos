Idea central

Un grafo no es solamente un dibujo.

Un grafo es una estructura matemática para representar relaciones. El dibujo es solo una visualización.

¿Qué es un grafo?

Un grafo tiene:

  • vértices o nodos;
  • aristas o conexiones.

Vocabulario mínimo

Vecinos

Nodos conectados directamente.

Grado

Cantidad de vecinos.

Camino

Secuencia de aristas.

Dirección

No dirigido

Si A se conecta con B, B se conecta con A.

Dirigido

La conexión tiene orientación: A → B no implica B → A.

Pesos

No ponderado

Solo importa si existe conexión.

Ponderado

La arista tiene costo: distancia, tiempo, riesgo, capacidad.

Disperso o denso

Un grafo disperso tiene pocas aristas comparado con el máximo posible.

Un grafo denso tiene muchas conexiones.

Esta distinción influye en la representación.

Lista de aristas

aristas = [
    ("A", "B"),
    ("A", "C"),
    ("B", "D"),
]

Simple para guardar conexiones; menos cómoda para consultar vecinos.

Lista de adyacencia

adyacencia = {
    "A": {"B", "C"},
    "B": {"A", "D"},
    "C": {"A"},
    "D": {"B"},
}

Natural para grafos dispersos y para consultar vecinos.

Matriz de adyacencia

    A  B  C  D
A [ 0, 1, 1, 0 ]
B [ 1, 0, 0, 1 ]
C [ 1, 0, 0, 0 ]
D [ 0, 1, 0, 0 ]

Útil cuando consultar si existe una arista debe ser directo.

NetworkX

NetworkX sirve para crear, analizar y visualizar grafos. No reemplaza nuestra implementación: la complementa.

import networkx as nx
G = nx.Graph()
G.add_node("A")
G.add_edge("A", "B")

Dibujar un grafo

import matplotlib.pyplot as plt
pos = nx.spring_layout(G)
nx.draw(G, pos, with_labels=False)
nx.draw_networkx_labels(G, pos)
plt.savefig("grafo_visual.png")

def convertir_a_networkx(grafo):
    G = nx.Graph()
    for vertice in ...:
        G.add_node(vertice)
        for vecino in grafo.vecinos(vertice):
            G.add_edge(vertice, vecino)
    return G

Pruebas que diseñarás

  • Ambas implementaciones dan los mismos vecinos.
  • Vértice sin vecinos.
  • `contiene_arista` en ambos sentidos.
