# Guía de Markdown técnico — Clase 16

Para cada hallazgo usa esta secuencia:

```markdown
## Contrato
Qué debe garantizar la función.

## Evidencia
Entrada mínima, comando y resultado observado.

## Riesgo
Qué comportamiento incorrecto permite.

## Corrección
Cambio localizado propuesto.

## Prueba
Cómo evitar la regresión.
```

Usa `NOTE` para contexto, `TIP` para estrategia, `IMPORTANT` para contratos, `WARNING` para errores probables y `CAUTION` para riesgos que obligan a detenerse.
