02 · Motivación

El costo oculto de pop(0)

Lista de Python

pendientes = ["A", "B", "C"]
actual = pendientes.pop(0)

Sale A, pero B y C deben desplazarse. Repetirlo cuesta O(n).

Cola ligada

A→B→C

Mover frente al sucesor cuesta O(1).

Mismo resultado observable no significa mismo costo.

05 · Invariantes

Los casos frontera definen la estructura

Vacía

frente = final = None
tamaño 0

Un nodo

frente is final
sin sucesor

Varios

Frente alcanza tamaño nodos; final es el último.

Al retirar el único: limpia ambos extremos y separa el nodo retirado.

07 · BFS

FIFO conserva las capas

capa 0: A
capa 1: B, C
capa 2: D, E
capa 3: F

La cola mantiene el orden no decreciente de capas.

Descubrimiento

A⇒BC⇒DE

Marcar al encolar evita duplicados.

08 · Predecesores

Guardar padres, no caminos completos

NodoPredecesor
ANone
BA
DB
FD

Reconstrucción

F ← D ← B ← A

Invertir:

A → B → D → F

La misma reconstrucción sirve para BFS, 0-1 BFS y Dijkstra.
09 · Lista doble y deque

Un enlace hacia atrás habilita ambos extremos

inicioA⇄B⇄Cfinal

Capacidad

Agregar y retirar por inicio o final en O(1).

Obligación

Anterior y siguiente concuerdan; el retirado pierde ambos enlaces.

Deque es el TDA; lista doble es la implementación de esta práctica.
10 · 0-1 BFS

Más aristas puede significar menor costo

Ruta corta

A → X

1 arista · costo 1

Ruta gratuita

A → B → C → X

3 aristas · costo 0

BFS minimiza saltos; 0-1 BFS minimiza la suma de pesos.
11 · Regla de extremos

Peso 0 al inicio · peso 1 al final

Relajaciones de 0-1 BFS

Solo una mejora estricta actualiza distancia y predecesor.

13 · Pruebas

Protege reglas, no solo ejemplos felices

Estructuras

  • FIFO y ambos extremos
  • vaciar y reutilizar
  • nodos retirados desconectados
  • tamaño consistente

Algoritmos

  • ciclo e inalcanzable
  • vecino implícito y no mutación
  • más aristas, menor costo
  • mejora posterior y peso inválido
