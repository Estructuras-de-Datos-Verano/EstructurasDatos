Mini laboratorio

Predice antes de revelar

Primero nombra objetivo, restricciones y operación dominante.

Caminos

BFS: mínimo número de aristas

Invariante

La cola procesa capas de distancia no decreciente.

Prueba distintiva

Dos aristas frente a tres; sin pesos, solo cuenta la cantidad.

Cola → retirar en orden de descubrimiento por capas.

Caminos

0-1 BFS: dos extremos, dos prioridades

Peso 0

Agregar al frente: conserva distancia.

Peso 1

Agregar al final: prioridad posterior.

Si aparece peso 2, frente/fondo ya no representa todas las prioridades.

Caminos

Dijkstra: menor distancia tentativa

A→B(8), A→C(2), C→B(1). El camino con más aristas cuesta menos.

Heap

Extrae la menor prioridad e inserta mejoras.

Contrato

Pesos no negativos: lo extraído vigente puede fijarse.

Límite

“Ninguno estudiado” también es una decisión

  1. Nombrar el contrato violado.
  2. Explicar el invariante que se pierde.
  3. Declarar el alcance sin alterar la entrada.

No convertir negativos a cero. No usar valor absoluto. No inventar un algoritmo.

Conexión global

Kruskal + Union-Find

Ordenar aristas; aceptar la siguiente barata solo si une componentes distintas.

Operación

Consultar componentes y fusionar.

Salida

V−1 aristas o señal de desconexión.

Contraste

Dijkstra no produce necesariamente un MST

Dijkstra

Minimiza rutas desde un origen.

Kruskal

Minimiza la suma de toda la red.

La prueba correcta obliga a ambas funciones objetivo a discrepar.

Dependencias

Kahn: disponibles significa grado cero

Una tarea entra a la cola cuando todos sus requisitos fueron retirados, no cuando simplemente se descubre.

Ingeniería

Reutilizar, validar, interpretar

Importar→Adaptar→Validar contrato→Ejecutar→Interpretar

Copiar cinco implementaciones crea cinco oportunidades de divergencia.

Evidencia

Una prueba debe distinguir alternativas

  • Más aristas, menor costo: BFS vs Dijkstra.
  • Peso 0 debe adelantarse: cola vs deque.
  • Red global frente a origen: Dijkstra vs Kruskal.
  • Dos requisitos: BFS vs Kahn.
  • Peso negativo: rompe el contrato de Dijkstra.

Cierre

Problema → operación → estructura → algoritmo

Los algoritmos se vuelven eficientes cuando la estructura de datos coincide con la operación dominante del problema.
