Nearest Smaller Values

Para cada posición, encontrar la posición más cercana a la izquierda con un valor menor. Si no existe, reportar `0`.

CSES: cses.fi/problemset/task/1645/

Ejemplo oficial

numeros:   2 5 1 4 8 3 2 5
respuesta: 0 1 0 3 4 3 3 7

Las posiciones están indexadas desde 1.

Lectura estratégica

  1. ¿Qué pide exactamente?
  2. ¿Cuál es la entrada?
  3. ¿Cuál es la salida?
  4. ¿Qué significa cercano a la izquierda?
  5. ¿Qué pasa si no existe?

Responde en `notebook.md`.

Primera idea: buscar hacia atrás

para cada posición i:
    mirar i-1, i-2, ...
    detenerse en el primer valor menor
    si no aparece: responder 0

Diagnóstico

Trabajo repetido

Volvemos a mirar valores anteriores muchas veces.

Peor caso

La solución ingenua puede acercarse a `O(n²)`.

Si un dato anterior ya no puede ser respuesta, no lo cargues para siempre.

Candidatos útiles

Al procesar un valor actual, algunos valores previos quedan dominados.

Si son mayores o iguales, no serán el menor cercano para futuros valores donde el actual está más cerca.

candidato = (valor, posición)
conservar solo lo que puede responder

Aparece una pila

Insertar candidato

Agregamos el valor actual.

Descartar

Quitamos candidatos recientes que ya no sirven.

Ejemplo visual

(2, 1)
(5, 2)
(1, 3)

Al llegar `1`, candidatos `5` y `2` no pueden responder para `1` ni para ciertos futuros valores.

La discusión exacta va en `notebook.md`.

Pseudocódigo

pila = []
para cada (posición, valor):
    mientras pila no vacía y tope.valor >= valor:
        desapilar
    si pila vacía: respuesta = 0
    si no: respuesta = tope.posición
    apilar (valor, posición)

La comparación clave

Para buscar menor estricto, los valores iguales no sirven.

Por eso el descarte usa `mayor o igual`.

Explica en `notebook.md` por qué `>` no basta.

Invariante

Después de procesar cada posición, la pila conserva candidatos útiles en un orden que permite encontrar el menor cercano desde el tope.

Este invariante debe explicarse en `discusion.md`.

Complejidad

Cada valor entra a la pila una vez y sale como máximo una vez.

Tiempo: O(n)Memoria: O(n)
