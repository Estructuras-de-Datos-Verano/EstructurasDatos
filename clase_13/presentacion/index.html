Árboles AVL

Balance continuo, rotaciones locales y el mismo orden BST.

Pregunta inicial

Si un BST puede convertirse en una lista, ¿cómo podemos impedirlo?

[!NOTE] La forma del árbol determina cuántos nodos recorremos.

Idea central

AVL sigue siendo un BST.
AVL mantiene pequeña la altura.
[!IMPORTANT] Paga trabajo local al insertar para acelerar búsquedas futuras.

Factor de balance

FB(nodo) = altura(izquierdo) - altura(derecho)
0: equilibrado
+2: carga izquierda
-2: carga derecha

¿Qué cambia al rotar?

Cambia

Raíz local, enlaces y altura.

Permanece

Valores, orden relativo e inorden.

Caso LL — Rotación derecha

Secuencia: 30, 20, 10 · FB(30) = +2 · nueva raíz local: 20.

LL — La rotación es local

Nueve inserciones; solo cambia el subárbol desbalanceado.

LL — Evolución completa

El árbol se mantiene AVL antes de continuar con nuevas inserciones.

LL — Decisión exacta

if balance > 1 and valor < nodo.izquierdo.valor:
    return self._rotar_derecha(nodo)
[!TIP] Carga izquierda + inserción a la izquierda del hijo izquierdo.

Caso RR — Rotación izquierda

Secuencia: 10, 20, 30 · FB(10) = -2 · nueva raíz local: 20.

RR — La rotación es local

La corrección no reorganiza el resto del árbol.

RR — Evolución completa

Alturas y factores se actualizan en cada regreso recursivo.

RR — Decisión exacta

if balance < -1 and valor > nodo.derecho.valor:
    return self._rotar_izquierda(nodo)
[!TIP] Carga derecha + inserción a la derecha del hijo derecho.

Caso LR — Izquierda + derecha

Secuencia: 30, 10, 20 · el pivote está en una posición interior.

LR — La rotación es local

Primero se prepara el subárbol; después se reduce su altura.

LR — Evolución completa

El inorden es idéntico antes, entre y después de las rotaciones.

LR — Decisión exacta

if balance > 1 and valor > nodo.izquierdo.valor:
    nodo.izquierdo = self._rotar_izquierda(nodo.izquierdo)
    return self._rotar_derecha(nodo)

Caso RL — Derecha + izquierda

Secuencia: 10, 30, 20 · el pivote vuelve a ser el valor intermedio.

RL — La rotación es local

El resto del árbol conserva padres, hijos y orden.

RL — Evolución completa

El balance se restaura antes de aceptar la siguiente inserción.

RL — Decisión exacta

if balance < -1 and valor < nodo.derecho.valor:
    nodo.derecho = self._rotar_derecha(nodo.derecho)
    return self._rotar_izquierda(nodo)

Un AVL que no necesita rotar

20, 10, 30, 5, 15, 25, 35 · todos los factores quedan en [-1, 1].

Varias rotaciones en una secuencia

El mantenimiento es continuo: aparecen RR, RL y LL.

BST vs. AVL — Misma secuencia

Mismo inorden; alturas y profundidades muy distintas.

¿Por qué sobrevive el BST?

A < x < B < y < C

Una rotación cambia quién es la raíz local, pero A, B y C permanecen en el intervalo que les corresponde.

[!IMPORTANT] Si el inorden cambia, la rotación está mal implementada.

Inserción AVL

  1. Insertar como BST.
  2. Actualizar altura.
  3. Calcular factor de balance.
  4. Clasificar LL, RR, LR o RL.
  5. Rotar si hace falta.
  6. Devolver la nueva raíz local.

Alturas correctas

nodo.altura = 1 + max(
    altura(nodo.izquierdo),
    altura(nodo.derecho),
)
[!WARNING] Inorden correcto no implica alturas correctas.

¿Qué ganamos?

Buscar: O(log n)
Insertar: O(log n)
Rotar: O(1)

La altura se mantiene proporcional a log n porque el balance se corrige continuamente.

Pruebas esenciales

  • LL, RR, LR y RL;
  • secuencia sin rotación;
  • varias rotaciones;
  • inorden antes y después;
  • alturas almacenadas;
  • duplicados y valores negativos.
