Concepto informal

La pregunta correcta no es solo qué guardo

La pregunta es: ¿qué quiero hacer muchas veces?
¿buscar? ¿agregar? ¿contar? ¿consultar vecinos? ¿extraer el mínimo?

Lista: una representación natural

datos = [17, 3, 42, 17, 8]

42 in datos      # True
100 in datos     # False
datos.append(5)  # agregar al final

Muy cómoda para

Guardar una secuencia, recorrerla, conservar orden y agregar al final.

Puede ser costosa para

Buscar un valor si no sabemos dónde está.

Búsqueda en lista

def buscar_en_lista(datos, objetivo):
    for valor in datos:
        if valor == objetivo:
            return True
    return False

Si el valor no aparece, hay que llegar al final para saberlo.

Conjunto: buscar se vuelve la operación natural

datos = [17, 3, 42, 17, 8]
valores = set(datos)

42 in valores     # True
100 in valores    # False

Ganamos

Búsqueda de pertenencia muy rápida.

Cambiamos

No conserva repeticiones y no se piensa como secuencia ordenada.

Experimento interactivo: lista vs conjunto

Conteo con diccionario

conteos = {}

for valor in datos:
    if valor not in conteos:
        conteos[valor] = 0
    conteos[valor] += 1

Idea

Usamos cada valor como llave y guardamos su frecuencia como valor.

$$\text{valor} \mapsto \text{frecuencia}$$

Conteo con Counter

from collections import Counter

conteos = Counter(datos)
conteos.most_common(5)

Uso repetido

Una estructura conviene más cuando la operación importante se repite.

$$\text{costo total} = \text{preparar datos} + \text{responder consultas}$$

Comparación rápida

Estructura Natural para Atención
list Secuencias, orden, recorrido. Buscar puede requerir revisar muchos elementos.
set Pertenencia y unicidad. No representa repeticiones como una lista.
dict Asociar llaves con valores. Hay que diseñar bien las llaves.
Counter Frecuencias. Es una especialización de diccionario.
Elegir una estructura de datos es elegir qué operaciones queremos hacer eficientes.

Idea: la implementación puede cambiar sin cambiar la forma en que se usa la estructura.

Ejemplo conceptual: pila

Una pila trabaja con el último elemento que entró.

Interfaz

push, pop, peek, is_empty.

Regla

Último en entrar, primero en salir.

Interfaz vs implementación

Interfaz

Lo que el usuario necesita saber para usar la estructura.

pila.push("dato")
ultimo = pila.pop()
vacia = pila.is_empty()

Implementación

Cómo se guarda internamente y cómo se logra cada operación.
