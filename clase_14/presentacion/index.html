Cola de prioridad y heap

FIFO

Sale quien llegó primero.

Prioridad

Sale quien tiene la clave prioritaria.

Min-heap

Padre ≤ hijos; mínimo en raíz.

Max-heap

Padre ≥ hijos; máximo en raíz.

Un heap no es un BST. Mantiene orden parcial, no recorrido ordenado.

Árbol completo en un arreglo

índice:       0  1  2  3  4  5
valor:       [1, 3, 5, 7, 6, 9]

padre(i)      = (i - 1) // 2
izquierdo(i)  = 2*i + 1
derecho(i)    = 2*i + 2

Árbol y arreglo son dos vistas del mismo estado.

Insertar y extraer

Insertar + sift-up

  1. Agregar al final.
  2. Comparar con padre.
  3. Intercambiar si es menor.
  4. Repetir hacia la raíz.

Extraer + sift-down

  1. Guardar raíz.
  2. Mover el último.
  3. Elegir hijo menor.
  4. Intercambiar hacia una hoja.
Cada reparación toca solo un camino de altura O(log n).

Last Stone Weight

Extraer repetidamente las dos piedras más pesadas. Para [2, 7, 4, 1, 8, 1], el resultado es 1.

heap = [-peso for peso in piedras]
heapq.heapify(heap)
mayor = -heapq.heappop(heap)
segundo = -heapq.heappop(heap)
heapq.heappush(heap, -(mayor - segundo))
