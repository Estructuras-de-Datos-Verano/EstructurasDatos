# Guía breve de Markdown

Usa títulos jerárquicos, párrafos breves, tablas para comparar y bloques de código con lenguaje. Una respuesta técnica debe afirmar, justificar y mostrar evidencia.

## Alertas de GitHub

> [!NOTE]
> Agrega contexto que conviene recordar.

> [!TIP]
> Ofrece una estrategia útil, no una respuesta completa.

> [!IMPORTANT]
> Señala una condición necesaria para que la entrega sea válida.

> [!WARNING]
> Advierte sobre un error probable, como confundir heap con BST.

> [!CAUTION]
> Reserva esta alerta para riesgos serios, como mezclar archivos de otro alumno.

## Checklist

- [ ] Pendiente
- [x] Completado
